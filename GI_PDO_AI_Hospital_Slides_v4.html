数据本质

一条训练样本是一个 PDO-drug pair：同一个 PDO 的 baseline 表达与某个药物配对，标签是该 PDO 对该药的药敏数值。

三张核心表

表行列 / 字段
expression_matrixPDOgene_1 ... gene_G，值为 log2(TPM+1) 或 normalized expression
drug_tabledrugdrug_name, SMILES, target, MoA
response_tablePDO-drug pairpdo_id, drug_name, AUC / inhibition_rate

模型输入 / 输出

输入： baseline expression vector + drug SMILES tokens
输出： scalar drug response，例如 AUC

# expression_matrix.csv pdo_id, TP53, EGFR, ERBB2, KRAS, ... PDO_001, 8.20, 4.71, 2.03, 6.45, ... PDO_002, 5.44, 6.18, 1.20, 7.90, ... # drug_table.csv drug_name, SMILES, target, MoA Oxaliplatin, "...", DNA damage, platinum chemotherapy 5-FU, "...", TYMS, antimetabolite # response_table.csv pdo_id, drug_name, AUC, inhibition_rate PDO_001, Oxaliplatin, 0.42, NA PDO_001, 5-FU, 0.63, NA PDO_002, Oxaliplatin, 0.77, NA

本队列应对齐的字段

pdo_id 必须在 expression、drug-response、临床信息、图像、药后 RNA 表中保持一致；drug_name 必须能映射到 SMILES / target / MoA。

数据本质

一条样本对应一个 microwell / drug-condition。每个 microwell 中约一组 organoids，记录药物、浓度、重复、viability，并生成 bulk-like RNA expression。

已发表实验设置

  • CRC organoids on SMARchip。
  • 4 个常见抗 CRC 药物。
  • 每药 8 个浓度、5 个重复。
  • 药物处理后获得 phenotype 与 transcriptome。

适合抽象成的 AI 任务

control / baseline state + drug + dose → treated RNA / pathway response / viability

任务 B：药后分子状态预测

07 / 16

问题定义

给定 PDO 的 control 表达和药物条件，预测药物处理后的转录组或通路响应。

为什么需要 subset 药后 RNA

  • 只测 baseline + AUC：只能做药敏预测。
  • 测 control + treated RNA：可以建模药物诱导的分子变化。
  • 结合 viability：可以分析分子响应如何解释药敏。
建议： 不必全队列全药物测药后 RNA。优先选择 20–40 例代表性 PDO、4–8 个 anchor drugs、1–2 个剂量、1–2 个时间点。

任务 C：图像时序响应预测

09 / 16

问题定义

利用非侵入式明场图像，预测药物处理后的生长抑制、死亡/破碎形态和耐药轨迹。

参考路径

  • OrganoID：brightfield / phase-contrast 下的单个 organoid 识别、标注和追踪。
  • deepOrganoid 类工作：brightfield image → CellTiter-Glo viability。
  • single-organoid resolution drug screening：单个 organoid 轨迹可区分 transient / persistent response。
关键价值： 图像让每个 well 内的 organoid 异质性进入模型，而不只是一个终点 viability 数值。

使用场景

当目标不仅是预测药敏，还希望解释 drug response 背后的调控状态时，可在 subset 上加入 ATAC 或蛋白读出。

推荐方式

同一个 PDO、同一药物条件、同一时间点下平行分样：

  • aliquot A → bulk RNA-seq
  • aliquot B → bulk ATAC-seq
  • aliquot C → proteomics / RPPA / targeted protein
  • 同条件保留 viability 和 image
不建议： 全队列一开始就铺三模态；更稳的是 10–20 例 × 2–4 个代表药物的机制验证层。

增强任务

可输出：differential accessibility、motif/TF activity、peak-to-gene response、regulatory biomarker。
可选增强层适合机制解释与高阶模型

执行优先级与最小可行数据包

优先级采集内容支撑任务说明
P0 必须baseline RNA-seq + 多药多剂量药敏 + DMSO + replicate任务 A没有这一层，无法稳定建立模型主任务。
P1 强烈建议明场时序图像 day0/24h/72h/120h任务 C体现 PDO 模型的表型与动态响应优势。
P2 建议WES / panel / CNV / MSI/MMR 等 genomic features任务 A / 解释提升药敏预测解释性和分型分析。
P3 subsetcontrol + drug-treated RNA-seq / targeted RNA任务 B用于药后状态预测，建议在代表性样本上完成。
P4 可选paired ATAC / protein / IF / spatial validation机制解释用于提升机制深度和验证模型发现。

最小可行数据包

100+ PDO： baseline RNA-seq + 8–20 drugs × 5–8 doses × ≥3 replicates + DMSO + endpoint viability + day0/24h/72h/120h brightfield。

高阶扩展： 20–40 PDO × 4–8 anchor drugs × 1–2 doses × 24h/72h × ≥2 replicates，采集药后 RNA 并与图像/viability 对齐。
执行优先级先保证可建模，再扩展机制层
